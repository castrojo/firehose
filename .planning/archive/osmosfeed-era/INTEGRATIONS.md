# External Integrations

**Analysis Date:** 2026-01-26

## APIs & External Services

**CNCF Landscape:**
- CNCF Landscape YAML API - Project metadata (names, descriptions, repo URLs)
  - URL: `https://raw.githubusercontent.com/cncf/landscape/master/landscape.yml`
  - Used in: `scripts/fetch-landscape-data.js`
  - Purpose: Match GitHub repos to official CNCF project names and descriptions
  - Auth: None (public GitHub raw content)

**GitHub Atom Feeds:**
- GitHub Releases Atom feeds - Release notes and changelog content
  - Pattern: `https://github.com/{org}/{repo}/releases.atom`
  - Used in: `scripts/extract-html.js` (build-time), `static/release-html-renderer.js` (runtime)
  - Purpose: Fetch release HTML content from GitHub repositories
  - Auth: None (public feeds)

**Project Blogs:**
- Various RSS/Atom feeds from CNCF project blogs
  - Examples: `https://blog.dapr.io/posts/rss`, `https://kubernetes.io/blog/rss`
  - Configured in: `osmosfeed.yaml` (60+ feed sources)
  - Purpose: Aggregate blog posts alongside release notes
  - Auth: None (public feeds)

**CORS Proxy:**
- AllOrigins API - CORS proxy for client-side feed fetching
  - URL: `https://api.allorigins.win/raw?url={encoded_feed_url}`
  - Used in: `static/release-html-renderer.js`
  - Purpose: Bypass CORS restrictions when fetching feeds from browser
  - Auth: None (public service)

## Data Storage

**Databases:**
- None (static site generator)

**File Storage:**
- Local filesystem only:
  - `public/cache.json` - Generated by osmosfeed, contains aggregated feed data
  - `public/landscape-data.json` - Generated by `fetch-landscape-data.js`, CNCF project metadata
  - `public/html-content.json` - Generated by `create-html-data.js`, cached HTML content
  - `public/*.html` - Generated static site files

**Client-side Storage:**
- localStorage:
  - Key: `firehose_feed_cache`
  - Purpose: Cache fetched feed content for 30 minutes
  - Used in: `static/release-html-renderer.js`

**Caching:**
- Build-time cache: `public/cache.json` (configured via `cacheUrl` in `osmosfeed.yaml`)
- Client-side cache: localStorage (30-minute TTL)
- No server-side caching infrastructure

## CI/CD & Deployment

**Hosting:**
- GitHub Pages
  - URL pattern: `https://{username}.github.io/{repo}/`
  - Deployment dir: `./public`
  - Triggered by: GitHub Actions workflow

**CI Pipeline:**
- GitHub Actions
  - Workflow: `.github/workflows/update-feed.yaml`
  - Triggers:
    - Push to `main` branch
    - Daily schedule (midnight UTC)
    - Manual via `workflow_dispatch`
  - Actions used:
    - `actions/checkout@v2` - Clone repository
    - `actions/setup-node@v2.1.4` - Setup Node.js v16
    - `peaceiris/actions-gh-pages@v3` - Deploy to GitHub Pages
  - Build command: `npm run build`
  - Auth: `GITHUB_TOKEN` (automatic GitHub Actions secret)

**Deployment Flow:**
1. Workflow triggers on schedule or push
2. Check out repository code
3. Install npm dependencies
4. Run build (fetch feeds, generate static files)
5. Deploy `./public` to `gh-pages` branch
6. GitHub Pages serves the site

## Feed Sources Configuration

**Feed Management:**
- Configured in: `osmosfeed.yaml`
- Total sources: 60+ RSS/Atom feeds
- Categories:
  - CNCF Graduated Projects (40+ sources)
  - CNCF Incubating Projects (20+ sources)
  - Main CNCF blog (1 source)

**Feed Types:**
- GitHub Releases Atom feeds (`.atom` extension)
- Blog RSS feeds (various paths: `/rss`, `/feed.xml`, `/blog/rss.xml`)

**Feed Processing:**
- Build-time: Osmosfeed fetches and parses all feeds into `cache.json`
- Build-time: `extract-html.js` enriches with HTML content from Atom feeds
- Runtime: `html-loader.js` and `project-info-loader.js` enhance display
